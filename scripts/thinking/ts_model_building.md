# 三维建模法

> **适用场景**：搭建分析框架、整理知识结构、理解复杂系统
> **来源**：td_model_taxonomy（模型分类三维论）+ st_mece_principle（MECE原则）+ td_cognitive_four_components（信息处理四器模型）
> **调用证据**：暂无 appliedIn 记录，来源为用户在 2023.05 发现的原创分类体系
> **编译时间**：2026-03-10

## 步骤

1. **判断模型类型**：当前问题适合哪种模型？
   - **流程模型**：事物在同一层如何流动？→ 流程图（考验经验总结）
   - **层级模型**：事物如何向上抽象/向下分解？→ 模块化/树形（考验范畴抽象）
   - **维度模型**：事物可沿哪些维度拆解？→ 四象限/星型/表格（考验维度拆解）

2. **MECE 检验**：
   - 各组之间相互独立不重叠？
   - 所有组完全穷尽不遗漏？
   - 四种常用方法：二分法、流程法、要素法、象限法

3. **叠加第二维**（如需要）：
   - 你的习惯组合：横向流程 × 纵向层级
   - 必要时补全第三维（维度拆解）

4. **四器完整性校验**：用信息处理四器扫一遍框架——
   - 感应器（感知/情绪输入）：有没有被忽略的输入？
   - 处理器（模型/认知加工）：加工逻辑是否完整？
   - 存储器（记忆/沉淀）：成果如何留存？
   - 效应器（行动/改变世界）：最终如何落地？

## 你的独特视角

大多数人只用一种模型（通常是流程图或层级图），你发现了第三维——维度模型。四象限图和表格法是维度模型的工具子类。三种模型考验不同能力：经验总结、范畴抽象、维度拆解。四器校验来自你对认知信息处理的理解——不是只有"处理器"重要，感应器包含共情和艺术，效应器才是真正改变世界的部分。
