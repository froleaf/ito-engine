# ITO Engine 架构

## 一句话

用户通过对话和笔记输入碎片化的思考，ITO 把它们编织成一张带有个人认知印记的知识图谱，并推动走向输出。

---

## 知识图谱：五层洋葱

所有数据存在一个 JSON-LD 文件里（`ontology/ontology.jsonld`）。你可以把它想象成五层同心圆：

```
┌─ 导航层 ─────────────────────────────────────────────────────┐
│  领域（DomainRoot）→ 分支（Branch）                           │
│  就像图书馆的楼层和书架，帮你找到东西在哪                       │
│                                                               │
│  ┌─ 知识层 ─────────────────────────────────────────────────┐ │
│  │  知识节点（KnowledgeNode）+ 学习材料（Material）          │ │
│  │  每个节点直接携带你的个人数据：                             │ │
│  │    理解深度 / 独特见解 / 认知歧义 / 输出潜力               │ │
│  │                                                           │ │
│  │  ┌─ 连接层 ────────────────────────────────────────────┐  │ │
│  │  │  跨域关联（CrossDomainLink）                         │  │ │
│  │  │  连接不同领域的知识，记录你发现的类比和同构             │  │ │
│  │  │                                                      │  │ │
│  │  │  ┌─ 产出层 ──────────────────────────────────────┐   │  │ │
│  │  │  │  知识沉淀（KnowledgeDeposit）                  │   │  │ │
│  │  │  │  思考沉淀（ThinkingDeposit）                   │   │  │ │
│  │  │  │  从碎片走向成型作品的中间态                      │   │  │ │
│  │  │  │                                                │   │  │ │
│  │  │  │  ┌─ 编译层 ────────────────────────────────┐   │   │  │ │
│  │  │  │  │  思维脚本（ThinkingScript）              │   │   │  │ │
│  │  │  │  │  从你的认知行为中提炼出的个人方法论        │   │   │  │ │
│  │  │  │  └─────────────────────────────────────────┘   │   │  │ │
│  │  │  └────────────────────────────────────────────────┘   │  │ │
│  │  └───────────────────────────────────────────────────────┘  │ │
│  └─────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

**关键特征**：知识节点不只是「某个概念的定义」——它同时记录了**你和这个概念的关系**。同一个「贝叶斯推理」节点，在你的图谱里带着你的理解深度、你的独特见解、你用它分析过什么问题。

## 核心数据流：从碎片到图谱

每次对话或笔记的处理都经过两个阶段：

```
你的自由表达（聊天、笔记、链接）
        │
        ▼
┌─ 阶段一：知识提取 ──────────────────────────────┐
│                                                   │
│  "你说的话里有什么？"                              │
│                                                   │
│  · 识别概念、理论、人物、作品                       │
│  · 捕捉你的原创观点和类比                          │
│  · 检测跨领域关联                                  │
│  · 发现你无意中使用的思维模型                       │
│  · 判断这次交互偏 Input / Think / Output           │
│                                                   │
│  输出：结构化的中间数据（不写任何文件）              │
└───────────────────────┬───────────────────────────┘
                        │
                        ▼
┌─ 阶段二：知识处理 ──────────────────────────────┐
│                                                   │
│  "这些东西该放进图谱的哪里？"                      │
│                                                   │
│  · 新概念 → 创建节点 + 补充同级知识                │
│  · 旧概念 → 更新深度、追加见解                     │
│  · 跨域发现 → 创建 CrossDomainLink                │
│  · 成型想法 → 创建 Deposit（沉淀产出）             │
│  · 认知歧义 → 标记，结束时温和指出                  │
│  · 写入交互记录 + 重建可视化                       │
│                                                   │
└─────────────────────────────────────────────────┘
```

## Skill 体系：谁调用谁

ITO 有 17 个 Skill，分为**用户直接使用**和**被其他 Skill 调用**两类。

```
用户直接使用                          内部调用（用户不直接触发）
─────────────                        ─────────────────────────
/ito ─── 主入口                      knowledge-extract ─── 提取
/chat ─── 聊天                       knowledge-process ─── 处理
/pass-note ─── 传笔记                 ontology-init ─── 初始化领域
/write-note ─── 写笔记                dormant-check ─── 沉寂检测
/bootstrap ─── 历史灌入               add-material ─── 材料录入
/plan ─── 精力规划
/weekly-review ─── 周度复盘
/review ─── 内容评价（只读）
/compile-thinking ─── 思维编译
/cold-start ─── 冷启动
/reset ─── 重置
/reload ─── 重建
```

**调用链**：

```
首次使用：
  /cold-start → ontology-init → 知识提取 → 知识处理

日常使用：
  /chat、/pass-note、/write-note → 知识提取 → 知识处理
                                                  ├→ 创建 Deposit
                                                  └→ 捕捉 Todo

历史灌入：
  /bootstrap → 知识提取 → 知识处理（不记 session，不触发计划）

周度循环：
  /weekly-review → 材料跟进 → Todo 盘点 → 产出盘点
                                              ├→ /plan（更新计划）
                                              └→ dormant-check

只读操作：
  /review → 评价内容（不动图谱）
  /compile-thinking → 编译思维脚本
```

## 记忆层：六种记忆

```
memory/
├── goal_memory.jsonl ──────── 你的 2-4 条主线目标（极少变动）
├── plan_memory.jsonl ──────── 每周的精力分配计划
├── session_memory/ ────────── 每次深度交互的记录（按周分文件）
├── milestone_memory.jsonl ─── 周度总结（成长叙事）
├── todo.json ──────────────── 待办清单（话题、阅读、深化、行动）
├── rlhf_memory.jsonl ──────── Agent 策略反馈（自我进化用）
└── preference_memory.jsonl ── 你的偏好（交互风格等）
```

**写入规则**：除了 `todo.json`（可修改）之外，所有 `.jsonl` 文件只追加不修改——像日志一样，保证数据安全。

## 输出体系：从想法到作品

```
一个想法的生命周期：

  对话中的灵感
       │
       ▼
  planned（有意向）──── 图谱中创建 Deposit 实体，还没有内容
       │
       ▼
  deposited（已沉淀）── 内容文件生成在 output/deposits/
       │
       ▼
  published（已发布）── 发到小红书 / 公众号 / 其他平台
```

每份 Deposit 内容文件分两部分：**AI 导读**（200-500 字摘要）和**原文**（你的完整内容）。Agent 默认只读导读以节省 token。

## 管理循环：PDCA

```
                    /weekly-review
                  ┌───────────────────┐
         Act      │  统计本周数据      │  Check
      ◀──────────│  知识饮食分析      │──────────▶
      调整行为     │  成长路径追踪      │  发现问题
                  │  材料 + Todo 盘点  │
                  │  产出盘点          │
                  └────────┬──────────┘
                           │
                           ▼
                        /plan
                  ┌───────────────────┐
         Plan     │  精力分配规划      │  Do
      ◀──────────│  I/T/O 均衡       │──────────▶
      制定计划     │  领域权重调整      │  日常交互
                  └───────────────────┘
```

## 文件系统总览

```
ito-engine/
│
├── ontology/
│   ├── ontology.jsonld ─────── 唯一的知识图谱（所有实体和关系）
│   └── _meta/
│       ├── domains.md ──────── 领域名称清单
│       └── thinking_scenarios.md  思维脚本路由表
│
├── memory/ ─────────────────── 六种记忆（见上文）
│
├── inbox/
│   ├── notes/ ──────────────── 待处理笔记（用户放入）
│   ├── bootstrap/ ──────────── 历史内容（灌入用）
│   ├── conversations/ ──────── 对话存档（Agent 写入）
│   └── processed/ ──────────── 已处理归档
│
├── output/
│   ├── deposits/ ───────────── 沉淀产出（Markdown 文件）
│   ├── reviews/ ────────────── 评价存档
│   └── drafts/ ─────────────── 草稿
│
├── scripts/
│   ├── build_visualization.py  图谱可视化生成脚本
│   └── thinking/ ───────────── 编译后的思维脚本
│
├── .claude/skills/ ─────────── 17 个 Skill（Prompt 文件）
├── templates/ ──────────────── 数据结构模板
├── _init/ ──────────────────── 初始状态快照（/reset 用）
├── visualization.html ──────── 知识图谱可视化（自动生成，勿手动编辑）
└── agent_profile.json ──────── Agent 配置
```
